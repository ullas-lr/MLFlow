Part 4 Model Monitoring

Understanding Drift

Concept & Types

1. Data Drift

Input Distribution Changes

Occurs when the statistical properties of the input data change over time, making the model's training data less representative of current reality.

Example: Seasonal Shift

  • Summer → Winter transition changes disease patterns
  • Query types shift from "sunburn" to "flu symptoms"
  • Demographic changes in patient population

2. Model Drift

Performance Degradation

Happens when the relationship between input variables and the target variable changes, causing the model's predictive performance to decline.

Example: Metric Decay

  • Quality Score drops from 0.92 → 0.72
  • Response Latency increases from 30s → 50s
  • Concept drift where medical guidelines change

Monitoring Imperative

Both Data Drift and Model Drift affect Edge and Cloud deployments equally. Continuous monitoring is essential for patient safety.
